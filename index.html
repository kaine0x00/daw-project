XSS

Cross-Site Scripting (XSS) is a security vulnerability that allows attackers to inject malicious scripts into web pages viewed by other users. This can enable attackers to steal data, hijack sessions, or trick users into performing unwanted actions. XSS typically occurs due to insufficient input validation, allowing scripts to be executed within the context of a trusted site.

Reflected XSS: Reflected XSS occurs when malicious code is injected into a URL and then reflected back in the response (e.g., in an error message or search result) without validation. It requires user interaction, like clicking a malicious link, for the attack to execute.

Stored XSS: Stored XSS (or persistent XSS) happens when the malicious script is saved on the server (e.g., in a database or message board). This script is then served to other users who visit the affected page, making it more dangerous, as it doesn’t rely on user interaction beyond viewing the page.

DOM-based XSS: DOM-based XSS is a client-side attack where the vulnerability exists in the document object model (DOM) itself. Malicious scripts manipulate the page content directly in the browser without needing to communicate with the server. This type is harder to detect since it doesn’t involve server-side data storage or reflection.

Self XSS: Self XSS is a form of social engineering where attackers trick users into running malicious code in their own browser console. It’s not an XSS vulnerability in the website itself but rather in how the site allows users to interact with it.

Hardware Hacking

Hardware hacking involves the process of modifying or manipulating electronic devices and their components to achieve a desired outcome, often to improve functionality, enhance performance, or exploit vulnerabilities. This can include everything from repurposing existing hardware to creating custom circuits or utilizing existing technology in unintended ways.

Bluetooth Hacking: Involves exploiting vulnerabilities in Bluetooth-enabled devices to gain unauthorized access or manipulate data. Techniques include sniffing traffic, executing man-in-the-middle attacks, or pairing with devices without permission.

Wi-Fi Hacking: Targets vulnerabilities in wireless networks, often using methods like packet sniffing, de-authentication attacks, or cracking WEP/WPA encryption to gain unauthorized access to network resources.

NFC Hacking: Involves manipulating or intercepting communications between NFC-enabled devices. Attackers may use techniques like cloning NFC tags, or conducting relay attacks to spoof or steal data.

Ethernet Hacking: Focuses on wired network vulnerabilities. Techniques include ARP spoofing, VLAN hopping, or exploiting weaknesses in network switches to intercept data packets or gain unauthorized access.

Rubber Duckies: Refers to a type of USB device that can be programmed to execute keystroke injection attacks. When plugged into a computer, it acts like a keyboard, executing pre-defined scripts to exploit vulnerabilities or extract data.

IDOR

Insecure Direct Object Reference (IDOR) is a type of access control vulnerability that allows an attacker to gain direct access to objects by modifying parameters in the URL or request. This can lead to unauthorized access to sensitive data or functionality, often due to insufficient validation of user input and lack of proper access controls.

Insecure Object References: This occurs when applications expose a reference to an internal implementation object, such as a file or database key. If these references are predictable, attackers can manipulate them to access unauthorized objects.

URL Manipulation: This type of IDOR occurs when an attacker modifies the URL parameters to access resources or data they are not authorized to view. For example, changing a user ID in the URL can allow an attacker to access another user’s account information.

Parameter Tampering: This involves altering the parameters of a request sent to a server to bypass security controls. Attackers may change the parameters to retrieve, modify, or delete resources they are not supposed to access.

Access Control Issues: IDOR often highlights flaws in an application’s access control mechanisms. If an application fails to properly enforce user permissions for accessing sensitive objects, it can lead to unauthorized data access and potential data breaches.

Phishing

Phishing is a type of cyber attack that aims to trick individuals into providing sensitive information, such as usernames, passwords, and credit card details, by impersonating a trustworthy entity. Phishing typically occurs through deceptive emails, websites, or messages, and it exploits human psychology rather than technical vulnerabilities.

Email Phishing: This is the most common form of phishing, where attackers send fraudulent emails that appear to be from legitimate sources. These emails often contain malicious links or attachments, prompting victims to enter personal information or download malware.

Spear Phishing: Unlike general phishing attacks, spear phishing targets specific individuals or organizations. Attackers gather personal information about their targets to create highly personalized messages that increase the likelihood of success.

Whaling Phishing: This variant of phishing targets high-profile individuals, such as executives or key decision-makers within an organization. The attacks are often more sophisticated, using tailored messages that appear legitimate and urgent to trick the victim into providing sensitive information.

Vishing Phishing: Voice phishing, or vishing, involves using phone calls to deceive victims into revealing sensitive information. Attackers often pose as legitimate entities, such as banks or tech support, and may use social engineering tactics to build trust.

Smishing Phishing: Smishing combines SMS messaging with phishing tactics. Attackers send text messages that appear to be from legitimate sources, often containing links to fraudulent websites or requests for personal information.

SQL Injection

SQL Injection is a code injection technique that exploits vulnerabilities in an application's software by allowing an attacker to interfere with the queries that an application makes to its database. This can enable attackers to view data they are not normally able to retrieve, modify or delete data, and in some cases, execute administrative operations on the database.

In-Band SQL Injection: This type of SQL injection is the most straightforward and common. It occurs when an attacker is able to use the same channel to both launch the attack and gather the results. This can include retrieving data through error messages or using UNION-based queries to get results directly.

Blind SQL Injection: In blind SQL injection, the attacker is unable to see the result of their query. Instead, they ask the database a true or false question and determine the answer based on the application's response. This can take longer but can still reveal sensitive information.

Out-of-Band SQL Injection: This type of attack is less common and occurs when an attacker is unable to use the same channel to launch the attack and gather results. Instead, they use different channels to retrieve data, often by triggering the database to send data to an external server controlled by the attacker.

Union-Based SQL Injection: Union-based SQL injection allows an attacker to combine the results of the original query with the results of another query, typically to extract sensitive data from the database. This technique is often employed when the attacker knows the structure of the database.

Malware

Malware is a broad term used to describe any malicious software designed to harm, exploit, or otherwise compromise a computer system or network. This includes a variety of harmful software types that can disrupt, damage, or gain unauthorized access to sensitive information.

Virus: A virus is a type of malware that attaches itself to legitimate programs or files. Once executed, it can replicate itself and spread to other files and systems, potentially causing damage and disrupting normal operations.

Worm: Unlike a virus, a worm is a standalone malware that replicates itself to spread to other computers, often exploiting vulnerabilities in network protocols. Worms can consume bandwidth and cause system failures without user intervention.

Trojan: A Trojan horse is a type of malware that disguises itself as a legitimate application or software. It tricks users into installing it, which then allows unauthorized access or control over the infected system.

Ransomware: Ransomware is a type of malware that encrypts files on a victim's computer, making them inaccessible until a ransom is paid to the attacker. This form of malware can lead to significant financial loss and data compromise.

Spyware: Spyware is malware that secretly monitors user activity and gathers personal information without their knowledge. It can track browsing habits, collect sensitive data, and often comes bundled with other software.

Adware: Adware is software that automatically displays or downloads advertisements when a user is online. While not always malicious, adware can degrade system performance and may collect user data for targeted advertising without consent.

Denial of Service (DoS) Attacks

Denial of Service (DoS) Attacks aim to make a service unavailable by overwhelming it with a flood of illegitimate requests, which can disrupt operations and deny access to legitimate users.

SYN Flood: This attack exploits the TCP handshake by sending numerous SYN requests to a target without completing the handshake, consuming server resources and potentially crashing the server.

UDP Flood: A UDP flood sends a large number of UDP packets to random ports on a target system. The system checks for applications listening on those ports and responds with ICMP packets, overwhelming its resources.

HTTP Flood: An HTTP flood attack targets web servers by overwhelming them with a massive volume of HTTP requests, consuming bandwidth and server resources, leading to service outages.

Zero-Day Exploits

Zero-Day Exploits are attacks that take advantage of software vulnerabilities that are unknown to the vendor, meaning there is no patch available to protect against them. These exploits are highly valuable in the cybercriminal world.

Remote Zero-Day: These exploits can be executed from outside the targeted network, allowing attackers to exploit vulnerabilities without physical access to the system.

Local Zero-Day: Local zero-days require physical access to the system to exploit the vulnerability, allowing attackers to execute malicious actions once inside.

Web Application Zero-Day: This type of zero-day targets vulnerabilities in web applications, potentially allowing attackers to gain unauthorized access or manipulate sensitive data.
